# Day 1 Exercise – Agile (User) Research Stories & Markdown

## Objective

Create a small project plan using research stories and document it in Markdown; where research stories are user stories, with a different focus and wording.

**NOTE:** _"research stories"_ is a concept local to this course matter, in general, there is only _"user stories"_.

## Task

Write 3–5 Research stories (RS):

- US Format: _As a [user], I want [goal], so that [reason]_
- RS Format: _Within [scientific field], investigate/experiment on/using [fact/design], to understand or corroborate assumption [phenomena/theory]_
  - being a structure of **area**, **exploration**, **manifestation**

## Example

```md
# Study Project

## User Stories (this should be Research Stories)

### As a student, I want to organise notes so that I can revise efficiently.

Implemented according to criteras;

- Notes from today should be highligted in any display
- etc.

### as a student, i want to compare different ways of organising notes, to improve methodology

Following criterias apply;

- possibility to store such organisations of notes, to use them later on
```

## Suggested Challenges

- Add priorities to each story
- Group stories into a sprint
