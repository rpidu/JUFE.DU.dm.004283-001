# AGILE Scientific Method – 1 Week Crash Course

## Executive Summary

This crash course introduces a practical, lightweight approach to the digital scientific method using only openly available resources from Wikipedia. Over five days, students learn how to plan work using Agile user stories, document using Markdown, manage work with Git and CLI, structure scientific reports using IMRAD, design scripts with the IPO model, and connect everything through the Scientific Method.

- Structure research using the [IMRAD format](https://en.wikipedia.org/wiki/IMRAD)
- Document work with [Markdown](https://en.wikipedia.org/wiki/Markdown)
- Track changes using [Version control](https://en.wikipedia.org/wiki/Version_control) and [Git](https://en.wikipedia.org/wiki/Git)
- Plan work using [Agile software development](https://en.wikipedia.org/wiki/Agile_software_development) and [User stories](https://en.wikipedia.org/wiki/User_story)
- Write scripts using the [Input–process–output model](https://en.wikipedia.org/wiki/IPO_model)

The course emphasizes **hands-on exercises**, reproducibility, and clear communication.
