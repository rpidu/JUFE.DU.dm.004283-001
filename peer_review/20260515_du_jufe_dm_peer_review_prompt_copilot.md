# Week 1 Learning Targets – Agile Scientific Method Crash Course

## Concepts

- What is Agile software development and how is it used to plan work?
- What is a user story / research story and how is it structured?
- What is Markdown and how is it used for documentation?
- What is a command-line interface (CLI) and why is it used?
- What is Git and version control and why are they important?
- What is the IMRaD structure and how is it used in scientific reporting?
- What is the scientific method and how does it structure inquiry?
- What is the IPO (Input–Process–Output) model in programming?
- What is an algorithm in the context of scripting?
- What is a hypothesis and how is it used in experiments?
- What is reproducibility in scientific work and why is it important?

## Challenges

### Student Work

- Can the student create and structure 3–5 research stories using the defined format?
- Can the student add priorities and organise research stories into a sprint?
- Can the student initialise a Git repository and perform commits?
- Can the student create branches, perform edits, and merge changes in Git?
- Can the student publish and share a repository online and clone another student repository?
- Can the student implement an IPO-based script in a programming language?
- Can the student extend the IPO script with multiple inputs, processing steps, and outputs?
- Can the student structure the IPO implementation as a function with comments and documentation?
- Can the student reflect on learning and document insights in the script?

### Group Work

- Can the group create and manage a shared repository with appropriate collaboration structure?
- Can the group coordinate branching and merging workflows across multiple students?
- Can the group produce and integrate a Markdown-based IMRaD report in the repository?
- Can the group define impact, theory, challenge, and tentative solution in the report?
- Can the group define hypotheses and null hypotheses from research questions?
- Can the group design an experiment and document method and reproducibility?

## Artifacts

### Student Artifacts

- Does the student repository contain structured research stories in Markdown?
- Does the student repository show evidence of commits, branching, and merging?
- Does the student repository include an IPO script file with documentation and comments?
- Does the student repository include shared or cloned peer work where required?

### Group Artifacts

- Does the group repository exist and is it publicly accessible with proper structure?
- Does the group repository include a central shared README.md using IMRaD structure?
- Does the IMRaD report include introduction, methods, results, and discussion sections?
- Does the report document hypotheses, experiment design, and reproducibility considerations?
- Does the repository reflect merged contributions from all group members?
